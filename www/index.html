Project Description

The EJB standard took a giant step between version 2.1 and 3.0 - the same step is (in our eyes) still missing in the JMS standard: To provide a simple to use, pure Java interface solution to remote calls... only asynchronous in this case. Even Spring has not a solution satisfying us, so we have started to develop the MessageApi as a remedy and just released version 1.0.

Motivation: The business code should concentrate on the message that is to be sent or what it has to do when a message is received. What technology is used, where the message is going or coming from, how the message is encoded, etc. is to be provided by a configurable adapter layer and injected into the business code.

We would like it to become a standard in some way (in some fits of megalomania I think it maybe even become JMS 2.0 :-), so we decided to make it publicly available so everybody can take a look at what we're up to. We welcome any feedback from feature requests to patches.
